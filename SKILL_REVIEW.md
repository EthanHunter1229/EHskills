## ⚠️ 需要改进的部分

### 1. 字号对照不完整
**问题：** `formatting_rules.md` 提到"字号对照：二号=22pt，三号=16pt，小三=15pt，四号=14pt，小四=12pt，五号=10.5pt，小五=9pt"，但文档中实际使用的"小五号"在 JSON 配置里写的是 9pt。

**建议：** 核对中文字号与磅值对应关系：
- 小五号 = 9pt ✅（footer 使用正确）

### 2. 三级标题字体问题
**潜在问题：** 楷体_GB2312 在某些系统上可能不存在（特别是 macOS 和非中国大陆 Windows）。

**建议：**
- 在脚本中添加字体回退机制
- 或在文档中注明依赖 `楷体_GB2312`，建议用户安装

**改进方案：**
```python
# 在 set_run_font 中添加回退逻辑
font_fallback = {
    "楷体_GB2312": ["楷体", "KaiTi", "STKaiti"],
    "黑体": ["SimHei", "STHeiti"],
    "宋体": ["SimSun", "STSong"]
}
```

### 3. 标题层级识别的边界情况

#### 问题 3.1：四级标题的正则过于宽松
**当前正则：** `"^\\d+[\\.、]"`

**问题：** 会误判这些文本为标题：
- "2021年公司营收..."
- "100.5万元..."

**建议改进：**
```json
"pattern": "^\\d{1,2}[\\.、]\\s*[\\u4e00-\\u9fa5]"
```
限制：1-2位数字 + 点/顿号 + 空格（可选）+ 至少一个汉字

#### 问题 3.2：封面标题识别过于严格
**当前逻辑：**
- 必须是第一个非空段落
- 或前4段内包含"招股说明书"且长度<80

**问题：** 如果文档开头有页眉/水印/空白页，可能误判。

**建议：** 在 SKILL.md 中明确说明需要用户清理封面前的无关内容。

### 4. 表格数字识别的正则问题
**当前正则：** `r"^[\d\.,%\-\+\(\)\s￥$]+$"`

**问题：**
- 会把"2021-2023"识别为数字（应该是）
- 但"￥"和"$"混用可能导致对齐混乱
- 缺少对千分位分隔符（逗号）、百分号的处理

**建议优化：**
```python
NUMBER_RE = re.compile(r"^[\d\.,，%％\-\+\(\)（）\s￥¥$]+$")
```
增加中文括号、中文逗号、中文百分号支持。

### 5. 页码的灵活性不足
**当前实现：** 只支持"第 X 页 共 Y 页"或"- X -"

**改进建议：**
- 增加罗马数字页码选项（目录、释义常用）
- 增加起始页码配置
- 在 SKILL.md 中说明封面/目录的页码需要单独处理

### 6. 缺少依赖说明
**问题：** 脚本需要 `python-docx`，但没有 `requirements.txt`

**建议添加：**
```txt
python-docx>=0.8.11
```

## 🐛 潜在 Bug

### Bug 1：自动编号计数器重置逻辑
**代码位置：** `format_docx.py:398-402`

**问题：**
```python
for lv, rank in level_rank.items():
    if rank > cur_rank:
        counters[lv] = 0
```
这个逻辑在遇到一个 h1 标题时，会重置所有更深层级（h2/h3/h4）的计数器。**这是正确的**。

但如果文档结构不规则（比如从 h1 直接跳到 h3，跳过 h2），h2 的计数器不会被使用，h3 会从 1 开始编号。**这是合理的行为**。

**结论：** 逻辑正确，但需在文档中说明"跳级标题"的编号行为。

### Bug 2：列表段落的字体颜色
**代码位置：** `format_docx.py:211-218`

**问题：** 列表段落会应用 body 样式，包括强制黑色。如果用户希望保留列表中的彩色文字（如红色警告、绿色成功），会被强制改为黑色。

**建议：** 在 SKILL.md 中说明"所有颜色会统一为黑色"，或增加配置项控制是否强制颜色。
